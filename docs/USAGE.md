# 使用文档 (User Manual)

## 1. 登录系统

访问管理后台（默认 `http://localhost:5173`），首次登录系统会提示设置初始管理员账号和密码。请妥善保管。

## 2. 概览 (Overview)

登录后进入概览页面，可以看到系统的实时运行状态：
*   **PV/UV/IP**: 实时访问量统计。
*   **请求分布**: 全球访问地理位置分布图。
*   **拦截统计**: WAF 拦截的恶意请求数量。
*   **爬虫统计**: 搜索引擎爬虫的访问情况。

## 3. 站点管理 (Site Management)

### 3.1 添加站点
点击“添加站点”按钮，填写：
*   **站点名称**: 唯一标识。
*   **域名**: 绑定的域名。
*   **端口**: 服务监听端口。
*   **模式**:
    *   **静态资源站**: 托管本地静态文件。
    *   **反向代理**: 代理到后端应用（如 localhost:3000）。

### 3.2 静态资源管理
对于静态资源站，点击“静态资源”按钮：
*   支持上传文件/文件夹。
*   支持创建文件夹。
*   支持批量删除。
*   支持在线解压 ZIP 文件。

### 3.3 预渲染配置 (SEO)
点击“渲染预热”按钮：
*   开启预渲染功能。
*   配置预渲染服务地址（如 Prerender.io 或自建服务）。
*   设置缓存时间。
*   手动触发页面预热。

### 3.4 推送配置
配置将新发布的 URL 自动推送到搜索引擎（如 Bing IndexNow）。

## 4. 防火墙 (Firewall / WAF)

在站点列表中开启防火墙功能，或在“防火墙”菜单中进行全局配置：
*   **规则引擎**: 开启 SQL 注入、XSS、命令注入等防护。
*   **IP 黑白名单**: 手动添加允许或禁止的 IP。
*   **地理位置控制 (GeoIP)**: 允许或禁止特定国家/地区的访问。
*   **CC 防护**: 设置请求频率限制。

### 4.1 GeoIP 说明
系统已集成在线 GeoIP API，无需额外配置即可识别访问者地理位置。系统会自动缓存查询结果以提高性能。

## 5. 日志查询 (Logs)

*   **访问日志**: 查看所有 HTTP 请求详情。
*   **拦截日志**: 查看被 WAF 拦截的请求详情及原因。
*   **爬虫日志**: 专门查看搜索引擎爬虫的访问记录。

## 6. 系统监控 (Monitoring)

查看系统的 CPU、内存使用率，以及 Redis 连接状态等健康指标。
